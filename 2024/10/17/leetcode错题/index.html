332.安排行程

有难度的，踩的几个坑，

  • 先把所有排列情况找出来，在在所有的情况中找出最靠前的，这个耗时太大了，不可取，
  • 然后是直接使用一阶段。想法是先将行程从小到大排序，因为递归时，是从1到n，前面的行程会先被访问，这样，只要找到了一种可行的行程，那么他就一定是最小的行程。
  • 这里需要不断递归，需要增删vector作为参数，要注意容器是否有效。

降低耗时采用的一些方式：

  • 传引用
  • 在for循环内层中，判断一下是否和上层循环的值一致，一致则continue继续循环

55.跳跃游戏

You are given an integer array nums. You are initially positioned at the array’s first index, and each element in the array represents your maximum jump length at that position.

Return true if you can reach the last index, or false otherwise.

虽然这个题也可以使用dfs+递归来完成，但这明显效率太低，应该选择其他算法。动态规划中对应的多个阶段，如果每个阶段都选择最优的，那么其实就是贪心算法，如果每个阶段选择的由max{g(x1) + f(s-x1)}中选择最大的，则是动态规划。

134.加油站

主要思路是，因为题上说最多只有一种解法。想象一个数组sub = gas - cost。所以起点必然是出现在连续正数中的第一个正数的位置。

比如，sub = [-2, -2, 2, -1, 7, 8, -3]，那么起点必然是2或者7对应的索引值（2， 4）。因为只能有一个解法，如何连续正数的第二个位置可以循环一周，那么因为第一个位置是正数，那么第一个位置比如可以循环一周，与题冲突了。

42.接雨水

分为两种思路，横着结算，纵着计算。

纵着计算：

对于每一个横坐标，找到其左侧和右侧的最高点，按照木桶效应，计算它们的最小值，然后减去当前坐标的高度，即为当前目标能够接到的雨水，将横坐标遍历一遍即可。关键是如何找到每一个坐标的左侧和右侧的最高点。比如找左侧最高点：从左到右变量横坐标，记录最高点，有更高的更新就行了。代码可见Trapping Rain Water

横着计算：

当左侧是一个下降的形状（不就可以用到单调栈吗），右侧出现上升的形状时，就找到了可以接雨水的那一层。

关键时递归中的几种情况：

  • 栈为空，只能push
  • 栈中元素更大，入栈，也能维护单调栈
  • 两者相等。这里注意必须要先出栈再入栈，因为在计算下一层的雨水值时，需要用到上一层的位置，这里必须更新。
  • 栈中元素更小时，则是一个循环，直到栈为空，或者栈中元素更大，这里需要使用上一层的位置，使用前必须判断栈不为空。

48.找最大柱形

和接雨水类似，最大柱形的高度必定是所有柱形的高度之一。所以可以从前往后遍历所有柱形，最大柱形的宽度即是当前柱形的最大宽度（找左边右边第一个小于他的柱形）。
